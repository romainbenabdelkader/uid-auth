# UID_AUTH Governance

Status: Draft governance principles for an open standard.

UID_AUTH is maintained by AUTHENTICA (France). This document defines
lightweight governance principles for the UID_AUTH standard. It does not
create a foundation, committee, certification body or legal authority.

## 1. Scope

UID_AUTH defines a sovereign, verifiable identifier for creative works, with
origin, integrity and AI / TDM rights signals. It provides a verifiable
technical reference only; the law, audit, the regulator or a court decide
legal qualifications.

UID_AUTH does not perform watermarking, fingerprinting, content detection,
DRM or rights enforcement.

## 2. Relationship to AURA

UID_AUTH and AURA are separate. AURA is an open, neutral standard for the
technical proof of origin and integrity; UID_AUTH is an identifier standard
that may use AURA-compatible concepts. Neither controls nor depends on the
other.

## 3. Open process

Changes should be proposed through public, written and traceable channels:

- GitHub Issues
- Pull Requests
- documented CHANGELOG entries

Private discussions may inform proposals, but normative changes should be
recorded publicly before being accepted.

## 4. Maintainer commitments

While AUTHENTICA remains the maintainer, it commits to:

- keep the standard open and freely implementable under the repository license;
- keep the identifier format and schema interoperable (JSON / JSON-LD);
- never extend the standard into enforcement, surveillance, DRM or watermarking;
- record normative changes publicly in the CHANGELOG.

## 5. Evolution

This document is a draft and may evolve. A future multi-stakeholder governance
model may be proposed.
